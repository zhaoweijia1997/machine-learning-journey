YOLOv8 GPU 性能对比总结
Intel Arc iGPU - OpenVINO 加速
==========================================================

模型规格对比:
┌──────────┬─────────┬──────────┬──────────┬───────────┐
│ 模型     │ 大小    │ 平均FPS  │ 延迟(ms) │ 相对速度  │
├──────────┼─────────┼──────────┼──────────┼───────────┤
│ yolov8m  │  50 MB  │  34.1    │  29.3    │  1.00x    │
│ yolov8l  │  84 MB  │  16.2    │  61.8    │  0.48x    │
│ yolov8x  │ 131 MB  │  10.2    │  97.8    │  0.30x    │
└──────────┴─────────┴──────────┴──────────┴───────────┘

测试配置:
- 设备: Intel Arc Pro Graphics (iGPU)
- 输入分辨率: 640x640 (固定)
- 并行推理: 4 个请求
- 优化模式: THROUGHPUT (吞吐量优先)
- OpenVINO 流: 4 个并行流

关键发现:
✓ yolov8m (medium) - 最佳平衡: 34.1 FPS, 适合实时应用
✓ yolov8l (large)  - 精度优先: 16.2 FPS, 可接受延迟
✓ yolov8x (xlarge) - 极限模型: 10.2 FPS, 最高GPU利用率

结论:
- yolov8x 是此GPU能运行的最大模型 (97.8ms延迟)
- 更大的模型意味着更高的GPU占用率
- 对于实时屏幕检测, 建议使用 yolov8m 或 yolov8l
- yolov8x 适合追求最高检测精度的离线场景
